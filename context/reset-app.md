## Context

I'm a 29-year-old software engineer (6 years at PayPay, Tokyo) currently back in Chandigarh, India for 15 days (April 28 – May 12, flight day). I have almost no obligations — just time, heat (40°C), and a desire to make this count. I want a React web app task manager deployed on GitHub Pages that tracks my progress across multiple life tracks.

The repo already exists with its own structure and style. Follow the existing patterns, component conventions, and design language already in the repo.

---

## What to Build

A task manager web app with the following characteristics:

- **Date range**: April 28 – May 12, 2026 (15 days, Day 15 is flight day — light tasks only)
- **Deployment**: GitHub Pages (static, no backend)
- **Persistence**: localStorage (no auth, single user)
- **Core UX**: Daily view showing tasks grouped by track, with completion tracking, a progress dashboard, and the ability to navigate between days
